Why it exists

You've finished something. You know the weeks it took, the drafts you cycled through, the sources you read. All of that is in your head.

Provenance puts it in a file. A tamper-evident file that lives next to your work, records your process as it happens, and can be exported as a PDF anyone can read.

What gets recorded

Provenance watches your project folder. Each file save gets an entry: a timestamp, a word count delta, and a git snapshot. Check-ins you write go in. Sources you log go in. Every paste into a tracked file, with where it came from.

File saves
Every time a tracked file changes, Provenance records a timestamp, word count, and a git snapshot. The whole history of the document, save by save.
Check-ins
A short field to note how a session went. Your own words, timestamped and chained into the ledger alongside the saves.
Sources
URLs, passages, and files you choose to log as references. Nothing is captured automatically — you add what belongs in the record.
Paste sources
When you paste into a tracked file, Provenance records the source URL or app, a 64-character preview, and a SHA-256 hash. Full content is never written to disk.
The authorship report

Export a PDF at any point. It shows the session timeline, the word count arc over time, every check-in you wrote, and the sources you logged. At the bottom: the hash chain head, so anyone reading it can verify the record was not edited after the fact.

Hand it to a publisher. Submit it with a grant application. Include it alongside a portfolio. It is a complete record of how something got made, in a format anyone can read.

Tamper-evident

Each entry in the ledger includes a SHA-256 hash derived from the entry before it. Change any entry retroactively and the chain breaks visibly. You can verify it yourself. The documentation explains how.

Provenance does not monitor this chain. It is a feature for honest makers, not a surveillance system.

Everything stays on your machine

No server. No account. No syncing through us. The ledger lives in a .ledger folder next to your project files, in plain Markdown and JSON you can open with any text editor.

Delete the folder at any time. Provenance forgets. Disconnect a project and the data stays on disk; the app stops watching. Export as Markdown, PDF, or a structured bundle for portability.

Pairs with Seed

Every project has a beginning. If yours started in Seed before it became something worth tracking, Provenance knows. The original note travels as a seed-trace — timestamped, versioned, attached to the first entry in the ledger. The full story of how a project came to be, from the first loose thought to the finished record.
